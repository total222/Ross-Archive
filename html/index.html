Unirse

El aprendizaje y arte que convergen en un solo sitio web.

Ross-Archive es una plataforma web dedicada a la preservacion de la cultura garifuna. Nuestro objetivo es brindar un espacio donde los usuarios puedan aprender, interactuar y compartir conocimientos sobre la rica historia y tradiciones garifunas. A traves de nuestro repositorio, foro, experiencias VR y traductores, buscamos fomentar el aprendizaje y la apreciacion de esta cultura unica. Te invitamos a explorar nuestro sitio y unirte a nuestra comunidad para contribuir a la preservacion y promocion de la cultura garifuna.

Ross-Archive es una aplicación web en la categoría de desarrollo de software con un propósito bien definido: ¡Entregarte una historia al alcance de tu bolsillo! Pensamos que esta iniciativa es necesaria porque la información es el recurso mas importante con el que contamos, y mantener las raíces de un grupo tan importante es una tarea que alguien debe hacer; nosotros creemos que esta es una solución efectiva capaz de escalar a mejoras innovadoras.

Nuestra historia

  • El inicio

    Se presenta un prototipo de diccionario garifuna en la feria tecno-académica Edutech en San Pedro Sula UTH.

  • Urgencia

    A las puertas de la feria tecnológica 2025 en Bethel, volvio a surgir la idea de extender y transformar el prototipo que habiamos presentado.

  • Nueva idea

    Surge un nuevo proyecto: Un traductor de verdad y un sistema de información.

  • Dominio

    Se decide que el proyecto sera una aplicación web, por lo que se compra un dominio de SquareSpace.

  • Prueba y error

    Se prueban distintos herramientas de traducción: Azure Custom traductor, Hugging transformers, Marian NMT, OpenNMT

  • Moses SMT

    Modelos de traducción basados en Redes Neuronales resultan en requerimientos de conocimiento avanzados y costes computacionales grandes, se opta por usar un traductor estadistico.

  • Lógica

    Por medio de la herramienta ClickUp, se escribe un documento con la lógica de los diferentes modulos, con el fin de establecer una guía para el desarrollo.

  • Referencias

    Se recopilan y organizan las referencias de diseño(páginas webs), librerias y recursos utilizados durante el desarrollo del proyecto.

  • Google Cloud Hosting

    Se usa una subscripción gratuita y se configura el hosting en Google Cloud para la aplicación, asegurando un entorno escalable y seguro.

  • Nginx

    Se implementa Nginx como servidor web.

  • Instalación y configuración de Moses

    Se instala y configura Moses SMT para la traducción automática, permitiendo la personalización y mejora del sistema.

  • Nginx, DNS y Dominio

    Se configura Nginx para manejar el tráfico web, se establece el DNS y se define un dominio para la aplicación.

  • HTTPS y SSL

    Se implementa HTTPS y SSL para asegurar la comunicación entre el cliente y el servidor, protegiendo la información sensible.

  • Landing Page

    Se diseña y desarrolla una landing page para la aplicación, optimizando la conversión de visitantes en usuarios.

  • SSH vsCode

    Se configura SSH para acceder de forma segura a los servidores y gestionar la infraestructura de manera remota.

  • Instalación de PostgreSQL en VM

    Se instala y configura PostgreSQL en una máquina virtual, asegurando un entorno de base de datos robusto y escalable.

  • Conectando el servidor PostgreSQL

    Se establece la conexión entre el servidor PostgreSQL y la computadora personal, permitiendo el acceso remoto a la base de datos.

  • Creación de tablas

    Se crean las tablas necesarias en la base de datos PostgreSQL, definiendo su estructura y relaciones.

  • Configurando PHP-fpm con Nginx

    Se configura PHP-fpm para trabajar con Nginx, mejorando el rendimiento y la gestión de la aplicaciones con PHP.

  • Configurando SSH con PostgreSQL

    Se configura SSH para acceder de forma segura a los servidores y gestionar la infraestructura de manera remota.

  • Instalación de Claude Sonnet 4.5 en terminal.

    Se instala Claude Sonnet 4.5 en la terminal y se configura Gemini CLI para mejorar la gestión de proyectos.

  • Buckets de Google Cloud

    Se crean y configuran buckets en Google Cloud Storage para el almacenamiento de datos.

  • Cuenta de Servicio para Buckets

    Se crea una cuenta de servicio en Google Cloud para gestionar los permisos de acceso a los buckets de almacenamiento.

  • Credenciales de Google Oauth2.0

    Se configuran las credenciales de Google Oauth2.0 para permitir la autenticación segura en las aplicaciones.

  • Desarollo de acceso, repositorio, cuenta, admin page, foro

  • Entrenamiento de Moses

    Se lleva a cabo el entrenamiento del equipo en el desarrollo frontend y se establece la conexión del servidor Moses con la página web.

Nuestros valores

Misión

La misión de Ross-archive es enseñarte sobre la etnia garífuna de una forma especial y profunda con la que puedas establecer una conexión con nuestro sitio y adquieras conocimiento sobre una de las etnias mas importantes de Honduras.

Visión

Nuestra visión es que con nuestro sitio web todos puedan conocer la historia e interesarse en ella, heredando este deseo y contribuir al progreso. ¡Por la historia, por el poder de aprender!

Objetivo

Nuestro objetivo es que Ross-Archive sea una plataforma web multifuncional que permita a los usuarios aprender, interactuar y compartir conocimientos sobre la cultura garífuna a través de diversas herramientas y recursos en línea.

Ross-Hub, el corazon de la informacion garifuna

Ross-archive cuenta con un repositorio orientado a la etnia garífuna que preserva textos, imágenes, videos y música con fácil acceso desde cualquier dispositivo. Nuestro repositorio esta diseñado con los parámetros internacionales de los repositorios digitales y cumple con todas las políticas respecto al deposito, acceso y manejo de la información de autores.
